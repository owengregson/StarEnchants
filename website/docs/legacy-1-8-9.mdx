# Legacy 1.8.9 support

The main StarEnchants jar is one universal download that runs **Paper 1.17.1 → 26.1.x and Folia**.
If your server is still on **Minecraft 1.8.9**, there is a **separate optional jar** — built from the
exact same engine and content — that runs there too.

:::info One engine, two jars
The 1.8.9 jar is **not** a different plugin: it is the whole StarEnchants engine compiled against the
1.8.8 server API and lowered to Java 8. Your `content/` configs, commands, items, and economy are
identical. Pick the jar that matches your server — never run both.
:::

## Requirements

| Need | Detail |
| --- | --- |
| Server | **Spigot / CraftBukkit / Paper 1.8.8 – 1.8.9** |
| Java | **Java 8** (the jar is lowered to Java-8 bytecode) |
| Everything else | optional — same as the modern jar |

## Install

1. **Download** the `StarEnchants-<version>-1.8.9.jar` asset from the
   [Releases](https://github.com/owengregson/StarEnchants/releases/latest) page — **not** the plain
   `StarEnchants-<version>.jar` (that one is the 1.17.1+ universal jar and will not load on 1.8.9).
2. Drop it into your 1.8.9 server's `plugins/` folder.
3. **Restart** the server. StarEnchants writes its default config into `plugins/StarEnchants/`, exactly
   like the modern jar.
4. Edit the configs, then run `/se reload`.

Everything in the [Concepts](./concepts/enchants.mdx), [Configuring](./configuring/index.mdx),
[Items](./items/enchant-books.mdx), and [Commands](./commands/index.mdx) docs applies unchanged.

## What is different on 1.8.9

1.8.9 predates several APIs the modern game added, so a few things degrade gracefully. None of this
needs any config change — the engine adapts automatically:

- **Newer materials fall back to their closest 1.8 equivalent.** A piece minted as `NETHERITE_HELMET`
  becomes `DIAMOND_HELMET` on 1.8 (1.8 has no Netherite); a handful of renamed sounds/particles map to
  their 1.8 names.
- **Off-hand-only effects do nothing** — 1.8 has no off-hand slot. Main-hand and armor work normally.
- **A few event-driven features run on a per-tick poll instead** (1.8 has no event for them): the
  ITEM_DAMAGE trigger, Heroic durability saves, and instant armour-swap detection. They work; the only
  edge is that swapping one enchanted piece for another of the *same* material is picked up when you
  close your inventory rather than the instant you do it.
- **Knockback control** is applied at the server's knockback-resistance level. Cancelling and reducing
  knockback are exact; amplifying it (a multiplier above 1) is best-effort.

Everything else — applying books/scrolls/dust, rendered lore, the combat effect engine, armor sets,
crystals, souls, slots, and the GUIs — behaves the same as on the modern jar, and is verified live on
a real 1.8.8 server on every change (see below).

## For maintainers: building, testing & releasing the 1.8.9 jar

The 1.8.9 jar ships **only when its live test gate is green** — there is no untested legacy lane.

**In CI (automatic).** Every push or PR that touches the engine runs the
[`Legacy 1.8.9` workflow](https://github.com/owengregson/StarEnchants/actions): it compiles
craftbukkit-1.8.8 on the runner via Spigot BuildTools, dual-compiles the whole plugin against it,
lowers it to Java 8, boots a **real 1.8.8 server under JDK 8**, and runs the reduced in-server smoke
suite (item apply + render, the no-PDC NBT-survival check, the material degrade, the combat spine, the
enchant GUI, and the degraded-feature checks). The release workflow runs the **same** gate and attaches
`StarEnchants-<version>-1.8.9.jar` to the GitHub release only when it passes — so the legacy jar can
never ship without a green Gate-4.

**Locally.** You need the 1.8.8 server jar in your local maven and a JDK 8:

```bash
# One-time: compile craftbukkit-1.8.8 into ~/.m2 (it is on no public repo).
java -jar BuildTools.jar --rev 1.8.8 --compile craftbukkit

# Build the downgraded 1.8.9 plugin jar → se/bootstrap/build/libs/StarEnchants-<ver>-legacy.jar
scripts/build-legacy-jar.sh bootstrap

# Run the live gate: build + downgrade the tester, boot real 1.8.8 under JDK 8, assert the smoke PASSes
scripts/legacy-smoke.sh
```

`scripts/legacy-smoke.sh` expects a JDK 8 at `~/se-legacy-buildtools/jdk8` and the
`craftbukkit-1.8.8.jar` produced by BuildTools (override the location with `SE_LEGACY_WORK`). The
design and seam map live in `docs/legacy-1.8.9-codeshare-design.md`.
